# Day 1｜AI 全域市場需求預測與動態商情分析

星期一上午八點四十五分，虛構的「跨企業產品決策中心」收到四封標題完全相同的郵件：請在今天下班前告訴我，下一季應先驗證哪一項需求。寄件人分別來自雜草町、義大開發、盛香珍X亙美質珍與寶島眼鏡。雜草町的營運主管林若晴想知道老宅夜間工藝、金工入門、羊毛氈與手捏陶，哪一類活動值得優先加開；義大的企劃經理陳柏宇面對購物、住宿與樂園套票，希望找到需求正在升溫、但又不會讓現場容量失控的方案；盛香珍的品牌團隊正在評估小泡芙新品主打方向，擔心只憑社群聲量會高估短期熱潮；寶島眼鏡的會員經理則要判斷商品、驗配服務與會員方案中，哪一項適合先做門市測試。

四位主管沒有交付真實營運資料，而是提供一份為教學建立的兩年期虛擬資料。每一列代表某家公司、某個商品或服務、某一個月份，欄位包含搜尋熱度、社群提及、網站造訪、促銷投入、相對價格、競品強度、節慶標記與模擬需求量。會議室裡很快出現三種意見。行銷同仁說搜尋熱度最高的就應該先做；財務同仁說先看促銷投入能不能帶回需求；營運同仁則提醒，如果預測錯十個單位與錯一百個單位，排班、備料與庫存承受的風險完全不同。資料分析師許安庭把白板分成三欄：要預測什麼、怎麼知道模型不是只記住過去、結果錯了會影響哪一個決策。

你的角色就是今天的資料分析顧問。你不能把虛擬數字包裝成企業事實，也不能看到高相關就宣稱某個行銷動作造成需求。你要先確認資料粒度與品質，再為小組負責的企業建立描述統計與趨勢圖；接著依時間先後切分訓練與測試資料，先用「上個月需求」做簡單基準，再建立可解釋的線性迴歸流程。下午的主管會議只接受一張決策卡，內容必須清楚寫出較佳方法、測試誤差、建議行動、限制與還要補蒐的真實資料。若證據不足，最專業的答案不是硬選一個商品，而是提出可控制規模、可停損、可再驗證的小型測試。今天的挑戰不是算出最漂亮的分數，而是讓另一位分析師明天可以重跑你的流程，也讓主管知道這個結果能支持什麼、不能證明什麼。

> 本日所有企業名稱僅作命題情境對照；CSV 為固定亂數種子建立的教學虛擬資料，不是企業真實紀錄。相關、迴歸係數與情境預測皆不可直接解釋為因果效果或正式營運承諾。

# Day 4｜AI 廣告精準投放與全通路行銷成效優化

第四天上午八點四十分，虛構的「跨企業產品決策中心」把一張一百萬元預算卡放在桌上（以下企業、資料與結論皆為虛構教學用途）。四個團隊都想搶資源：雜草町主張短影音互動最高，預算應集中 Meta；義大開發認為 Google 的高意圖訪客最接近套票成交；盛香珍X亙美質珍相信 YouTube 能累積新品記憶，不能只看短期訂單；寶島眼鏡則提醒 LINE 與 Email 規模較小，卻可能更適合服務提醒與回訪。各自拿出 CTR、轉換率、ROAS、觸及或互動，卻沒有人先核對分母、漏斗位置與最終價值。

營運主管要求你在下午五點前交兩份成果：一張百萬元全通路預算表，以及一張能執行、能停止、也能驗證增量的投放實驗卡。資料分析師提供三項線索。第一，六千列資料涵蓋通路、受眾、素材、裝置、曝光、點擊、造訪、轉換、花費、營收與簡化毛利率，可重建漏斗；第二，真正轉換事件稀少，只看 accuracy 可能掩蓋模型幾乎沒抓到正例；第三，不同門檻會交換 false positive 與 false negative，哪種錯誤較昂貴必須由業務、財務與治理設定，不能交給模型自動決定。
